# 30分程度の音声ファイルから曲を調査し、可能ならYouTubeリンクも出力する方法

## 結論

30分前後の録音ファイルから流れている曲を特定したい場合は、**音声ファイルを主資料、文字起こしを補助資料**として扱うよう明示して依頼するのが最も重要です。

また、**録音環境の情報は入れたほうがよい**です。特に以下の情報があると、推定精度が上がります。

- 何を録音したものか
  - 例: レコード再生、DJミックス、店内BGM、ラジオ、ライブ会場、クラブ
- 録音方法
  - 例: スマホのボイスレコーダーで空間録音、ライン録音、マイク録音
- 混入ノイズの有無
  - 例: 会話、環境音、車内音、アナウンス、操作音
- 曲のつながり方
  - 例: フル再生に近い、途中で切れる、複数曲がクロスフェードしている
- 音楽の傾向
  - 例: 日本語ラップ中心、90s R&B中心、ハウス中心

これらを書いておくと、AIが「聞き取りづらさ」や「誤認しやすい条件」を先に織り込めます。

---

## 依頼時の基本方針

依頼文には、次のルールを含めると安定します。

1. **音声を最優先で参照する**
2. **文字起こしは補助情報として使う**
3. **曲ごとの開始時刻と終了時刻を出す**
4. **曲名・アーティスト名・必要ならバージョン名まで出す**
5. **確信度や根拠も添える**
6. **YouTubeリンクはWeb検索で1件ずつ確認して付ける**
7. **不確実な曲は未確定として分ける**

---

## 推奨する依頼テンプレート

以下をそのまま使えます。

```markdown
あなたは音楽識別リサーチャーです。

目的:
約30分の録音ファイルの中で再生されている全ての曲を特定し、可能であれば各曲に対応するYouTubeリンクも付けて一覧化してください。

入力:
1. 音声ファイル（約30分）
2. 文字起こしテキスト

録音環境:
- これは[例: スマホのボイスレコーダーで空間録音したレコード再生音]です
- [例: 周囲の会話や環境音が混ざる可能性があります]
- [例: 曲が途中から録音されている場合があります]
- [例: フル再生ではなく、切り替えやクロスフェードがあるかもしれません]
- [例: ジャンルは日本語ラップ中心です]

作業ルール:
1. 音声ファイルを主資料として扱い、文字起こしは補助資料として使ってください
2. まず録音内で曲が流れている区間をすべて抽出してください
3. 各区間ごとに、歌詞・フレーズ・声質・曲調・ビート・固有名詞などから曲名とアーティストを推定してください
4. 必要なら曲のバージョン名、ミックス名、ライブ版かどうかも区別してください
5. 曲ごとに開始時刻と終了時刻を出してください
6. 確定度を「確定 / 有力 / 候補」のように分けてください
7. 根拠を簡潔に書いてください
8. Web検索で各曲に対応するYouTubeリンクを1件ずつ確認して付けてください
9. 不確実なものは無理に断定せず、候補として残してください

出力形式:
| No | 開始 | 終了 | 曲名 | アーティスト | バージョン/補足 | 確定度 | 根拠 | YouTube |
|----|------|------|------|--------------|------------------|--------|------|---------|
| 1 | 00:00:00 | 00:03:12 |  |  |  |  |  |  |

追加ルール:
- まず全区間を洗い出してから、各曲を順番に再確認してください
- 最終的に、確定曲一覧と未確定候補一覧を分けてください
- YouTubeリンクは、公式音源・公式MV・アーティスト公式チャンネルを優先してください
- 公式が見つからない場合は、最も妥当な視聴リンクを付けてください
```

## さらに精度を上げるために入れるとよい情報

### 1. 録音対象が何か
かなり重要です。

- レコード再生
- DJプレイ
- 自分のプレイリスト
- 店舗BGM
- ラジオ番組
- クラブイベント
- ライブ映像の音声

これで「曲が途中で切れるのが普通か」「MCが入るか」「同じ曲の別ミックスが出やすいか」が変わります。

### 2. ジャンルや年代
候補絞り込みに効きます。

- 日本語ラップ
- 90年代ヒップホップ
- UKソウル
- K-POP
- アニソン
- 2000年代J-POP

### 3. 録音品質
AIに期待値を合わせさせるのに有効です。

- 音がこもっている
- 小さい音量で録っている
- 会話が重なっている
- 低音が割れている
- スマホをポケットに入れたまま録音した

### 4. 曲数の目安
30分で何曲くらい入りそうかを書いておくと、区間分割が安定します。

- フル尺中心なら 6〜10曲程度
- 短く切り替わるDJミックスなら 10曲以上の可能性あり

## 実務上のコツ

### 音声を主、文字起こしを従にする
文字起こしは歌詞を誤認しやすく、会話やノイズも文章化してしまいます。
そのため、

- 区間検出: 主に音声
- 曲名推定: 音声 + 文字起こし
- 最終確認: Web検索

という順番がよいです。

### いきなり曲名確定まで求めず、段階的にさせる
1回で全部やらせるより、次の2段階にすると精度が上がりやすいです。

1. まず曲が流れている区間一覧を作る
2. その区間表をもとに曲名とYouTubeリンクを確認する

### 不確実な結果を分ける
全部を断定させると誤りが増えます。なので、

- 確定
- 有力
- 候補
- 未特定

のように分ける指示が有効です。

## 最小限版の依頼文

短く依頼したい場合はこれでも十分です。

```markdown
添付した30分程度の音声ファイルと文字起こしを使って、録音内で再生されている全曲を特定してください。
音声を主資料、文字起こしを補助資料として扱ってください。
曲ごとに開始時刻・終了時刻・曲名・アーティスト名・必要ならバージョン名を出し、根拠と確定度も付けてください。
可能であれば各曲のYouTubeリンクをWeb検索で1件ずつ確認して付けてください。
不確実な曲は未確定候補として分けてください。
録音はスマホのボイスレコーダーによる空間録音で、会話や環境音が混ざる可能性があります。
```

## まとめ

今回のような用途では、依頼文に **録音環境の説明を入れるのは有効** です。
特に重要なのは次の3点です。

1. **何を録音した音声か**
2. **どんなノイズや混入があるか**
3. **音声を主、文字起こしを補助にすること**

この3点を明示すると、曲調査の精度と出力の安定性がかなり上がります。
